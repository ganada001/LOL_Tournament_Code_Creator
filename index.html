📌 소개 (Introduction)

본 도구는 Riot Games Tournament API (V5)를 활용하여 사설 대회의 토너먼트 코드를 쉽고 빠르게 생성해 주는 프로그램입니다.

주요 특징:

  • 🔒 보안 강화: 사용자의 API Key 입력 없이 안전하게 동작 (Google Apps Script 백엔드 사용)
  • ⚡ 자동화: 클릭 한 번으로 토너먼트 생성부터 코드 발급까지 완료
  • 📢 디스코드 연동: 발급된 코드를 디스코드 채널로 자동 전송

🔍 화면별 상세 가이드 (Screen-by-Screen Guide)

메인 화면 (Main View)

프로그램 실행 시 가장 먼저 만나는 화면입니다.

  • 프리셋 버튼: 등록된 대회를 클릭하면 즉시 토너먼트 코드가 생성되고, 설정된 디스코드 웹훅으로 정보가 전송됩니다.
  • 상태 표시줄: 현재 백엔드(Stub/Production) 연결 상태를 실시간으로 보여줍니다.

일반 설정 (General Settings)

프로그램의 핵심 작동 환경을 설정합니다.

  • Stub API 스위치: 실제 데이터 소모 없이 테스트할 수 있는 '연습 모드'입니다. 기능 테스트 시에는 이 모드로 확인하는 것을 권장합니다.
  • Provider 등록: Riot API와 통신하기 위한 고유 공급자 ID를 발급받고 관리합니다.

수동 생성 (Manual Generation)

프리셋에 등록되지 않은 일회성 대회를 위한 기능입니다.

  • 대회 이름, 맵 형태(협곡/칼바람), 픽 방식 등을 직접 선택하여 즉석에서 코드를 생성할 수 있습니다.

프리셋 관리 (Preset Management)

자주 진행하는 대회 형식을 미리 저장해두는 곳입니다.

  • 디스코드 웹훅: 각 대회별로 결과를 전송할 디스코드 채널 주소를 다르게 설정할 수 있습니다.
  • 간편 추가/삭제: 여러 개의 대회 운영안을 생성하고 관리하기 편리합니다.

🚀 사용 방법 (How to Use)

  1. GitHub 저장소 또는 위 다운로드 버튼을 통해 프로그램을 다운로드합니다.
  2. 필요한 라이브러리를 설치합니다: pip install -r requirements.txt
  3. gui_main.py를 실행합니다.
  4. [설정] 탭에서 대회 프리셋(이름, 웹훅 URL 등)을 설정합니다.
  5. 메인 화면에서 대회 버튼을 누르면 코드가 생성되고 디스코드에 전송됩니다.

⚙️ 환경 및 공급자 설정 (Environment & Provider)

1. Stub API vs Production API

프로그램 상단의 Stub API 사용 스위치를 통해 실행 환경을 변경할 수 있습니다.

  • Stub API (테스트용): 실제 게임 데이터에 영향을 주지 않고 토너먼트 코드 생성 로직을 테스트할 수 있는 라이엇 제공 테스트 환경입니다. 승인 전 개발 단계에서 권장됩니다.
  • Production API (라이브 서버): 실제 리그 오브 레전드 라이브 서버에서 작동하는 코드를 생성합니다. 라이엇으로부터 Production Key 승인을 받은 후 사용 가능합니다.

2. 공급자 및 토너먼트 (Provider & Tournament)

토너먼트 코드를 생성하기 위해서는 먼저 Provider ID가 필요합니다.

  • Provider ID: 대회를 주최하는 주체를 식별하는 ID입니다. 설정 탭에서 'Provider 새로 생성' 버튼을 클릭하여 발급받을 수 있습니다.
  • 데이터 유지: 한 번 발급받은 ID는 프로그램 내부에 안전하게 저장되어 재실행 시에도 유지됩니다.
  • 주의사항: Stub 모드에서 발급받은 ID는 라이브 모드에서 호환되지 않으므로, 모드 변경 시 ID를 새로 생성해야 합니다.

🚀 빠른 시작 가이드 (Quick Start Guide)

프로그램의 핵심 기능을 빠르게 확인하시려면 다음 순서를 권장합니다.

  1. 프로그램 실행 후 하단의 [설정] 버튼을 클릭합니다.
  2. Stub API 사용이 활성화되어 있는지 확인합니다 (기본값: On).
  3. Provider 새로 생성 버튼을 눌러 테스트용 ID를 발급받습니다.
  4. 메인 화면으로 돌아와 생성된 프리셋 버튼을 클릭하여 실제 토너먼트 코드가 생성되는지 확인합니다.

* Stub 모드이므로 실제 게임 클라이언트에 입력 시 '유효하지 않은 코드'로 뜰 수 있으나, API 응답 자체는 표준 규격을 100% 준수합니다.

🛡️ 보안 및 시스템 아키텍처 (Security & Architecture)

본 프로그램은 라이엇 게임즈의 보안 가이드를 최우선으로 준수하도록 설계되었습니다.

  • API Key 미노출: 클라이언트 코드나 로컬 설정 파일에 Production API Key를 절대 저장하지 않습니다.
  • 중계 서버(Proxy) 방식: 모든 요청은 Google Apps Script 백엔드를 거쳐 전달되며, API Key는 서버측 환경 변수에서만 관리됩니다.
  • 데이터 비저장: 서버는 요청을 전달(Forwarding)할 뿐, 어떠한 유저 데이터나 대회 정보도 DB에 기록하지 않습니다.

❓ 자주 묻는 질문 (FAQ)

Q. 왜 직접 API 키를 넣지 않나요?

보안 때문입니다. 사용자 기기에 API 키가 남을 경우 유출 위험이 크기 때문에, 안전한 백엔드를 통해서만 통신하도록 제작했습니다.

Q. 생성된 코드가 디스코드에 오지 않아요.

[설정 - 프리셋 관리] 탭에서 유효한 Discord Webhook URL이 입력되어 있는지 확인해 주세요.

Q. 공식 승인 전에는 어떻게 테스트하나요?

프로그램에 내장된 'Stub API' 모드를 사용하면 승인된 키가 없어도 모든 기능을 미리 체험해 볼 수 있습니다.
